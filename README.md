# Amazon-sales-data-analysis
End-to-end Amazon sales data analysis using Python, SQL, and Power BI to analyze revenue, orders, category performance, fulfilment efficiency, and sales trends through interactive dashboards.
# Amazon Sales Data Analysis Project

# Project Overview
This project presents an end-to-end **Amazon Sales Data Analysis** aimed at understanding sales performance, order trends, category-wise revenue, fulfilment efficiency, and cancellation impact.  
The project demonstrates real-world Data Analyst skills using Python, SQL, and Power BI.

# Project Objectives
- Analyze total sales and order performance
- Identify top-performing product categories
- Track monthly sales trends
- Evaluate order status (Shipped, Cancelled, Returned)
- Compare Amazon vs Merchant fulfilment
- Build an interactive Power BI dashboard for business insights

# Dataset Information
- Dataset Name: Amazon Sale Report  
- Records: 120,000+ orders  
- Type: Transactional sales data  
  
# Tools & Technologies
- Python (Pandas, NumPy) – Data cleaning & preprocessing  
- SQL – Data querying and aggregation  
- Power BI– Dashboard creation & visualization  
- Excel / CSV – Data storage  

# Data Cleaning & Preparation
- Removed duplicate records  
- Handled missing and null values  
- Converted date columns to datetime format  
- Created derived columns (Month, Year)  
- Standardized category and status values  
- Ensured correct data types for numerical fields  

# Key KPIs
- Total Sales: ₹78.59 Million  
- Total Orders: 120K  
- Average Order Value: ₹652.88  
- Cancelled Orders %: 15.23%  

# Power BI Dashboard
# Overview

# Key Insights
- Sets and Kurtas generate the highest revenue
- Sales peak in April, followed by a gradual decline
- Majority of orders are successfully shipped
- Amazon fulfilment accounts for ~70% of total orders
- Cancellations contribute to measurable revenue loss

- Plan promotions around peak sales periods
